# 虚拟购物系统搭建方案

## 📋 项目概述

### 目标
将原JavaScript实现的虚拟购物系统集成到EPhoneS Android项目中,实现完整的AI驱动商城功能。

### 核心原则
1. **数据隔离**: 所有数据库和数据模型放在`aidata`模块
2. **模块解耦**: 新建`shopping`模块,仅依赖`aidata`和`core`模块
3. **桌面集成**: 在`desktop`模块添加商城入口图标

---

## 🏗️ 架构设计

### 模块依赖关系
```
app
 ├─ shopping (新建)
 │   ├─ aidata (数据层)
 │   └─ core (工具层)
 ├─ desktop
 └─ aidata
```

### 技术栈
- **UI**: XML布局 + Fragment
- **数据库**: Room Database
- **架构**: MVVM + Repository模式
- **图片加载**: Glide
- **依赖注入**: Hilt

## 📦 第一步: aidata模块 - 数据层搭建

### 1.1 数据库Entity创建

#### 文件位置: `aidata/src/main/java/com/susking/ephone_s/aidata/data/local/entity/`

**需要创建的Entity文件:**

1. **ShoppingCategoryEntity.kt**
   - 商品分类表
   - 字段: id(主键), name(分类名称), timestamp(创建时间)

2. **ShoppingProductEntity.kt**
   - 商品主表
   - 字段: id(主键), name, price(默认价格), description, imageUrl, categoryId(外键), variations(款式JSON), timestamp

3. **ShoppingVariationEntity.kt**
   - 商品款式表(可选,也可用JSON存储)
   - 字段: id(主键), productId(外键), name, price, imageUrl

4. **ShoppingCartItemEntity.kt**
   - 购物车表
   - 字段: id(主键), productId(外键), quantity, selectedVariationIndex, timestamp

5. **ShoppingOrderEntity.kt**
   - 订单历史表(用于礼物记录)
   - 字段: id(主键), chatId, items(JSON), total, recipients(收礼人JSON), timestamp

### 1.2 TypeConverter创建

#### 文件位置: `aidata/src/main/java/com/susking/ephone_s/aidata/data/local/entity/`

**ShoppingConverters.kt**

### 1.3 DAO接口创建

#### 文件位置: `aidata/src/main/java/com/susking/ephone_s/aidata/data/local/dao/`

**需要创建的DAO文件:**

1. **ShoppingCategoryDao.kt**
   - `getAll()`: 获取所有分类
   - `insert(category)`: 插入分类
   - `delete(categoryId)`: 删除分类
   - `findByName(name)`: 根据名称查找

2. **ShoppingProductDao.kt**
   - `getAll()`: 获取所有商品
   - `getByCategoryId(categoryId)`: 根据分类获取

4. **ShoppingOrderDao.kt**

## 📝 第五步: 实现细节补充

### 5.1 AI生成功能实现细节

#### GenerateShoppingItemsUseCase逻辑:

**输入参数:**
- chatId: 当前聊天ID
- categoryCount: 要生成的分类数量
- productCount: 每个分类的商品数量

**处理流程:**
1. 获取聊天上下文(角色人设、长期记忆、世界书、最近对话)
2. 获取已有分类列表
3. 构建System Prompt:
   - 强制AI复用已有分类名称
   - 要求返回纯JSON格式
   - 每个商品必须包含2-4个款式
4. 调用AI API(优先使用副API)
5. 解析JSON响应
6. 增量添加到数据库:
   - 检查分类是否存在,存在则复用ID
   - 生成商品图片URL(使用Pollinations AI)
   - 批量插入商品

**图片生成:**
- 使用Pollinations AI: `https://image.pollinations.ai/prompt/{英文prompt}`
- 商品主图: `product.image_prompt`
- 款式图: `variation.image_prompt`

### 5.2 购物车功能实现细节

#### 加入购物车逻辑:

**无款式商品:**
- 检查购物车是否已存在相同productId且无款式的商品
- 存在: 增加数量
- 不存在: 新增购物车项

**有款式商品:**
- 检查购物车是否已存在相同productId且相同款式的商品
- 存在: 增加数量
- 不存在: 新增购物车项

#### 结算流程:

**单聊模式:**
1. 用户点击结算按钮
2. 弹出确认对话框
3. 确认后创建礼物消息
4. 将购物车中已勾选的商品添加到礼物消息
5. 发送到聊天历史
6. 清空购物车中已结算的商品

**群聊模式:**
1. 用户点击结算按钮
2. 打开收礼人选择弹窗
3. 显示群成员列表(排除自己)
4. 用户选择收礼人
5. 确认后创建礼物消息(包含recipients字段)
6. 插入隐藏的系统提示消息(告知AI收礼人)
7. 发送到聊天历史
8. 清空购物车中已结算的商品

### 5.3 商品管理功能实现细节

#### 管理模式切换:

**切换状态:**
- isProductManagementMode: Boolean标志
- 切换时重新渲染商品列表

**管理模式UI变化:**
- 商品卡片上显示编辑和删除按钮
- 隐藏"加入购物车"按钮
- 删除分类按钮显示/隐藏(仅在非"全部"标签时显示)

#### 商品编辑:

**编辑现有商品:**
1. 点击编辑按钮
2. 打开ProductEditorFragment
3. 预填充商品数据(包括所有款式)
4. 用户修改
5. 保存时更新数据库

**添加新商品:**
1. 点击添加按钮
2. 打开ProductEditorFragment(空白表单)
3. 用户填写信息
4. 添加多个款式
5. 保存时插入数据库

### 5.4 款式管理功能实现细节

#### 动态款式输入:

**添加款式:**
- 点击"添加款式"按钮
- 动态创建款式输入块(包含名称、价格、图片上传)
- 添加到RecyclerView

**删除款式:**
- 每个款式块有删除按钮
- 点击时从RecyclerView移除

**保存款式:**
- 遍历所有款式输入块
- 提取数据并验证
- 保存到商品的variations字段

#### 款式选择:

**显示逻辑:**
- 商品有款式: 显示款式选择按钮
- 商品无款式: 直接加入购物车

**选择流程:**
1. 打开款式选择弹窗
2. 显示所有款式(RadioButton组)
3. 选择款式时更新价格和图片
4. 调整数量
5. 确认后加入购物车

## 🔧 第十一步: 潜在问题和解决方案

### 问题1: 图片加载慢

**原因:**
- Pollinations AI服务器在国外
- 网络延迟较高

**解决方案:**
- 使用Glide缓存策略
- 设置占位图和加载动画
- 考虑使用CDN加速

### 问题2: AI生成失败

**原因:**
- API配置错误
- 网络问题
- API限流

**解决方案:**
- 添加详细的错误日志
- 提供友好的错误提示
- 支持重试机制
- 使用副API作为备用

### 问题3: 购物车同步问题

**原因:**
- 多个Fragment共享购物车数据
- 数据更新不及时

**解决方案:**
- 使用SharedViewModel
- 使用LiveData/Flow观察数据变化
- 确保数据更新时通知所有观察者

### 问题4: 款式管理复杂度高

**原因:**
- 动态添加/删除款式
- 数据收集和验证

**解决方案:**
- 使用RecyclerView管理款式列表
- 封装款式数据类
- 统一验证逻辑

### 问题5: 群聊收礼人选择

**原因:**
- 需要获取群成员列表
- 需要区分用户自己

**解决方案:**
- 从ChatMessage获取群成员信息
- 使用originalName作为唯一标识
- 过滤掉用户自己
